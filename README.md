# 🍪 SnackEnglish

**Telegram-бот для мікронавчання англійської мови — по 2-3 хвилини на день, без перевантаження, прямо в месенджері.**

> Замість класичних курсів, які вимагають годин зосередженої уваги і які легко покинути, SnackEnglish пропонує "перекуси знань" — короткі, регулярні сесії у застосунку, який користувач і так відкриває десятки разів на день.

## 💡 Ідея проєкту

Класичні курси англійської мови вимагають багато часу й дисципліни — саме тому люди часто кидають навчання на півдорозі. **SnackEnglish** вирішує цю проблему через концепцію **"snack-навчання" (мікронавчання)**: користувач отримує невеликі, добре структуровані порції матеріалу щодня, без відчуття перевантаження.

Бот адаптується під рівень користувача (A1–C2) і пропонує щоденні слова, тести, тексти для перекладу та міні-курси граматики — усе в звичному інтерфейсі Telegram.

## ✨ Функціонал

### 📚 Навчання
- **Онбординг з вибором рівня** — після `/start` користувач обирає рівень (A1–C2), і весь контент підлаштовується під нього
- **Щоденні слова** — 5 нових слів на день з перекладом, транскрипцією та аудіоозвучкою
- **Міні-тести** — інтерактивні тести з 4 варіантами відповідей
- **Тексти для перекладу** — короткі адаптовані тексти з кнопкою показу перекладу
- **Снек-курси граматики** — покрокові міні-курси (наприклад, дієслово "To Be", часи): екран теорії → міні-тест → наступний крок
- **Тематичні словники** — подорожі, їжа, бізнес

### 🧠 Розумна ротація контенту
- Масиви `seenWords`, `seenTests`, `seenTexts` у моделі користувача виключають повтори
- Запити через оператор `$nin` гарантують видачу лише нового контенту
- Автоматичне скидання історії, коли контент рівня закінчується — без помилок і збоїв

### 🏆 Гейміфікація та утримання
- **Streaks 🔥** — трекінг днів активності поспіль
- **XP-система** — бали за кожне слово та правильну відповідь
- **Глобальний лідерборд** — щотижневий турнір із топ-3 переможцями та топ-10 рейтингом
- **Реферальна система** — запрошення друзів з нагородами
- **Cron-задачі** (`node-cron`, `setInterval`) — автоматичне підбиття підсумків турніру, обнулення балів, розсилка нагадувань

### 💎 Freemium та монетизація
- Безкоштовна версія з денними лімітами
- Premium: безлімітне навчання, розширені граматичні курси, детальна статистика
- Оплата через **Telegram Stars (XTR)** — нативні платежі в 1 клік без сторонніх систем

### 🛠️ Адмін-панель
- Статистика користувачів, масові розсилки з урахуванням лімітів API
- Управління Premium-користувачами (список, пагінація, картки з детальною інформацією, відкликання підписки з повідомленням)
- **ШІ-пайплайн генерації контенту**: чіткі constraint-based промти для ChatGPT/Claude, вбудований парсер (Regex + валідація), що автоматично розкладає згенерований контент по колекціях MongoDB

## ⚙️ Технологічний стек

| Категорія | Технологія | Причина вибору |
|---|---|---|
| Мова | **TypeScript** | Строга типізація, менше багів, зручний рефакторинг |
| Середовище | **Node.js** | Продуктивність та екосистема для Telegram-ботів |
| Bot Framework | **grammY** | Сучасніший і швидший за застарілі альтернативи (`node-telegram-bot-api`) |
| База даних | **MongoDB + Mongoose** | Гнучкість NoSQL для прогресу користувачів, масивів слів і статистики |
| Інфраструктура | **Hetzner VPS** | Надійний і бюджетний хостинг для продакшену |
| Платежі | **Telegram Stars (XTR) API** | Нативна монетизація без сторонніх платіжних систем |
| Планувальник | **node-cron** | Автоматизація тижневих турнірів та нагадувань |

## 🏗️ Архітектура

```
Користувач (Telegram)
        │
        ▼
   grammY Bot Layer  ──▶  Middleware (auth, session, error handling)
        │
        ▼
   Services Layer
   ├── progressService   (XP, streaks, ротація контенту)
   ├── seasonService      (турніри, лідерборд)
   ├── sentenceService     (вправи на побудову речень)
   └── adminService        (парсинг, розсилки, статистика)
        │
        ▼
   MongoDB (Mongoose Models)
   ├── User (рівень, XP, streak, seenWords[], premium...)
   ├── Word / Test / Text
   ├── SentenceExercise
   └── Season / Winners
```

Розробка велася поетапно — від базової механіки видачі контенту до повноцінної системи утримання, монетизації та автоматизації контенту через ШІ.

## 💰 Монетизація

Із зростанням аудиторії до **600+ користувачів** з'явилася потреба покривати витрати на сервер. Було впроваджено модель **Freemium**:

- **Free** — денні ліміти на слова та тести
- **Premium** — безлімітне навчання + розширений контент + детальна статистика

Оплата реалізована через нативний **Telegram Stars (XTR)** — користувачі можуть оформити підписку або задонатити проєкту в один клік, без виходу з месенджера.

## 🐛 Технічні виклики та рішення

| Проблема | Рішення |
|---|---|
| Telegram блокував бота при масових розсилках (rate limit) | Асинхронні паузи `await new Promise(res => setTimeout(res, 50))` — не більше 20-30 повідомлень/сек |
| Бот падав, якщо користувач заблокував його (403 Forbidden) | Глобальний `bot.catch` — тихе ігнорування/видалення заблокованих юзерів без крашу |
| `parse_mode: 'Markdown'` ламав рендеринг через спецсимволи (`_`, `'`) | Перехід на безпечний HTML-рендеринг у критичних місцях |
| Повтори контенту після зростання бази | `$nin`-запити + масиви `seenWords/seenTests/seenTexts` + авто-скидання циклу |
| `ConflictingUpdateOperators` у Mongoose | Оптимізація логіки оновлення документів, перехід на `returnDocument: 'after'` |
| Race condition у нарахуванні прогресу | Складені (compound) індекси в MongoDB + рефакторинг `progressService` |

## 🌿 Git Workflow

Розробка велася з дотриманням структурованого підходу до гілок і комітів:

**Гілки:**
- `main` — стабільна продакшн-гілка
- `feature/<назва>` — нова функціональність (наприклад, `feature/admin-add-content`, `feature/infinite-xp`, `feature/learning-menu`)
- Зміни потрапляють в `main` через **Pull Request** з обов'язковим рев'ю перед мерджем

**Коміти** (стиль [Conventional Commits](https://www.conventionalcommits.org/)):
- `feat: ...` — нова функціональність (напр. `feat: add referral system with rewards and sharing flow`)
- `fix: ...` — виправлення багів (напр. `fix: replace deprecated 'new: true' with 'returnDocument: after' in mongoose`)
- `chore: ...` — технічні задачі без зміни поведінки (напр. `chore(db): add compound indexes and fix race condition in progressService`)

Такий підхід дозволяє чітко відстежувати історію змін, легко ревертити окремі функції та підтримувати читабельний changelog.

## 🚀 Встановлення та запуск

```bash
# Клонування репозиторію
git clone https://github.com/MishaTomash/snackenglish.git
cd snackenglish

# Встановлення залежностей
npm install

# Налаштування середовища
cp .env.example .env
# Заповнити: BOT_TOKEN, MONGODB_URI, тощо

# Запуск у режимі розробки
npm run dev

# Продакшен-збірка
npm run build
npm start
```

## 📁 Структура проєкту

```
src/
├── models/          # Mongoose-схеми (User, Word, Text, TestQuestion, SentenceExercise, Season)
├── services/        # Бізнес-логіка (progress, season, sentence, admin)
├── handlers/         # Обробники команд і сесій grammY
├── keyboards/        # Inline/reply клавіатури
├── middlewares/       # Auth, error handling
├── utils/            # Парсери, форматери, допоміжні функції
├── jobs/             # Cron-задачі (тижневі турніри, нагадування)
└── seed.ts           # Наповнення бази верифікованим контентом
```
